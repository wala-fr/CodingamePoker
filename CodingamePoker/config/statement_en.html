Let's play poker! This is a port of the Texas hold'em version.
The source code of this game is available on this GitHub repo.

  Rules

The rules of Texas hold 'em can be found here.
The number of player varies from 2 to 4 .
The action moves clockwise around the table.
If only one player has still chips or after 600 rounds, the game is over. If the last hand is not over, it's cancelled.
The players are ranked by stack.
The card's abbreviations are here.
A list of how some special cases are dealt with is here.

ACTIONS

Possible actions

There are 5 possible actions :
  • CALL
  • CHECK
  • BET amount The amount corresponds to the number of chips that the player adds to the pot.
  • ALL-IN
  • FOLD

Action replacement

In some cases if you give an "invalid" action, it will be replaced by a "legal" one :
For example : BET 500 with a stack of $400 will be considered as an ALL-IN ...
The complete list can be found here.
The replacement informations are given in the console.

BLINDS

The small blind and big blind initial values are given in the inputs.
Then the level is increased every fixed number of hands and the blinds (small and big) are multiplied by a coefficient (those values are also given in the inputs).

Others

Victory Conditions
  • You own all the chips.
  • After 600 round you've got the biggest stack.
Loss Conditions
  • You've lost all your chips.
  • You do not respond in time.

  Game Input

Initialization input
Line 1: playerPositionX
Line 2: playerPositionY the coordinates of your pawn.

Input for one game turn
Line 1: opponentPositionX
Line 2: opponentPositionY the coordinates of the opponent's pawn.
Line 3: opponentLastRemovedTileX
Line 4: opponentLastRemovedTileY the coordinates of the last tile removed by the opponent ( -1 -1 if no tile has been removed (first round)).
Output
A single line containing the coordinates where you want to move your pawn, followed by the coordinates of the tile you want to remove.
Example: 1 4 7 4
You can also add a message :
Example: 1 4 7 4;MESSAGE
NB : You can print RANDOM instead of the 4 coordinates. Then a random possible move and tile will be chosen.
Constraints
2 ≤player number≤ 4
Response time first turn is ≤ 1000 ms.
Response time per turn is ≤ 50 ms.
round≤ 600
